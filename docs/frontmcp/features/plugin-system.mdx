FrontMCP's plugin system lets you add cross-cutting capabilities — caching, logging, context extensions, and more — without modifying your tools or resources.

## What Plugins Can Do

| Capability | Example |
| --- | --- |
| Register tools, resources, prompts | Dynamically add capabilities at startup |
| Add providers | Inject services into the DI container |
| Extend context | Add `this.remember`, `this.cache` to tool contexts |
| Hook into flows | Intercept request lifecycle stages |

## Quick Example: Audit Plugin

```ts
@Plugin({ name: 'audit', description: 'Logs all tool executions' })
class AuditPlugin extends DynamicPlugin {
  async onRegister(ctx: PluginRegistrationContext) {
    ctx.hooks.register({
      flow: 'tools:call-tool',
      stage: 'execute',
      type: 'did',
      handler: async (flowCtx) => {
        console.log(`Tool executed: ${flowCtx.toolName}`, flowCtx.output);
      },
    });
  }
}
```

Register it on your server:

```ts
@FrontMcp({
  info: { name: 'Server', version: '1.0.0' },
  apps: [MyApp],
  plugins: [new AuditPlugin()],
})
class Server {}
```

## Official Plugins

| Plugin | Description |
| --- | --- |
| **Cache** | Transparent caching for tool and resource responses |
| **Remember** | Session memory with `this.remember.get/set` and tool approval tracking |
| **CodeCall** | AgentScript runtime for safe, sandboxed AI code execution |

## Adapters

Adapters dynamically generate tools, resources, and prompts from external sources:

```ts
import { OpenAPIAdapter } from '@frontmcp/adapters';

@App({
  id: 'api',
  name: 'API',
  adapters: [
    new OpenAPIAdapter({
      url: 'https://api.example.com/openapi.json',
    }),
  ],
})
class ApiApp {}
```

The OpenAPI adapter reads your spec and generates one tool per operation automatically.
