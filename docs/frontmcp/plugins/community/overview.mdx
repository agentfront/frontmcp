## Community Plugins

The FrontMCP plugin system is designed to be extensible. Community members can create and publish plugins that add new capabilities to any FrontMCP server.

<Note>
  No community plugins have been published yet. Be the first to contribute!
</Note>

## Finding Community Plugins

<Steps>
  <Step title="Search npm">
    Search for packages tagged with `frontmcp-plugin` or `mcp-plugin`:
    ```bash
    npm search frontmcp-plugin
    ```
  </Step>
  <Step title="Check GitHub Topics">
    Browse repositories tagged with [`frontmcp-plugin`](https://github.com/topics/frontmcp-plugin) on GitHub.
  </Step>
  <Step title="Join the Community">
    Visit the [FrontMCP Discussions](https://github.com/agentfront/frontmcp/discussions) to discover and share plugins.
  </Step>
</Steps>

## Publishing Your Plugin

### 1. Create Your Plugin

Follow the [Create a Plugin](/frontmcp/guides/create-plugin) guide to build a plugin using the `@Plugin` decorator and `DynamicPlugin` base class.

### 2. Package It

```json package.json
{
  "name": "@yourscope/frontmcp-plugin-myfeature",
  "version": "1.0.0",
  "description": "FrontMCP plugin for MyFeature",
  "keywords": ["frontmcp", "frontmcp-plugin", "mcp", "plugin", "myfeature"],
  "peerDependencies": {
    "@frontmcp/sdk": "^0.4.0"
  }
}
```

### 3. Add Documentation

Include a README with:
- Installation instructions
- Configuration options
- Usage examples
- API reference

### 4. Publish

```bash
npm publish
```

## Plugin Guidelines

### Quality Standards

- TypeScript types for all exports
- Comprehensive error handling
- Unit tests with >80% coverage
- No security vulnerabilities

### Maintenance

- Respond to issues within 7 days
- Keep dependencies updated
- Follow semantic versioning
