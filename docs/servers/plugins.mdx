title: Plugins
slug: servers/plugins
---

Plugins extend apps with cross‑cutting capabilities like caching, metrics, or rate limiting. Plugins can contribute providers and global tool hooks, and they may augment metadata.

Cache plugin
- Adds transparent caching for tool outputs keyed by input
- Stores: memory or Redis
- Augments tool metadata with:
```
cache: true | { ttl?: number; slideWindow?: boolean }
```
- App usage example (conceptual):
```
@FrontMcpApp({
  id: 'expenses',
  plugins: [CachePlugin],
  tools: [...]
})
```

Options
- Memory: `{ type: 'memory', defaultTTL?: number }`
- Redis client: `{ type: 'redis-client', client, defaultTTL?: number }`
- Redis config: `{ type: 'redis', config: { host, port, password?, db? }, defaultTTL?: number }`

Hooks
- Plugins can register hooks for tool stages (will/did/around) to implement cache hit/miss, metrics, redaction, etc.

Tips
- Prefer explicit per‑tool `cache` configs for important endpoints
- Use short TTL + slideWindow for frequently accessed, frequently changing data
