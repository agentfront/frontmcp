The FrontMCP Dashboard Plugin gives you a real-time graph of every app, tool, prompt, resource, and agent in your server without writing instrumentation code. It ships an HTML UI powered by React Flow and connects via SSE so you always see the latest state.

## Installation & quick start

Install the plugin with:

```bash
npm install @frontmcp/plugins
```

### DashboardApp (recommended)

Add the `DashboardApp` to your `FrontMcp` configuration so it runs as a standalone scope:

```ts title="apps/dashboard.ts"
import { FrontMcp, LogLevel } from '@frontmcp/sdk';
import { DashboardApp } from '@frontmcp/plugins';

@FrontMcp({
  info: { name: 'Monitoring Server', version: '1.0.0' },
  apps: [DashboardApp],
  logging: { level: LogLevel.VERBOSE },
})
export class DashboardServer {}
```

Start your server and browse to `http://localhost:3000/dashboard`. The `DashboardApp` can see every scope because it runs with `standalone: true` and shares the `GraphDataProvider`.

### DashboardPlugin (advanced)

Want more control over middleware or routing? Register the plugin directly:

```ts
import { FrontMcp, LogLevel } from '@frontmcp/sdk';
import { DashboardPlugin } from '@frontmcp/plugins';

@FrontMcp({
  info: { name: 'Monitoring Server', version: '1.0.0' },
  logging: { level: LogLevel.INFO },
  plugins: [
    new DashboardPlugin({
      enabled: true,
      basePath: '/dashboard',
    }),
  ],
})
export class DashboardServer {}
```

## Configuration

| Option           | Default                    | Description                                                                    |
| ---------------- | -------------------------- | ------------------------------------------------------------------------------ |
| `enabled`        | `auto`                     | Enables the dashboard in development and disables it by default in production. |
| `basePath`       | `/dashboard`               | Base path for the HTML page, SSE stream, and MCP tools.                        |
| `auth.enabled`   | `false`                    | Toggle token authentication.                                                   |
| `auth.token`     | â€”                          | Secret token that clients must supply while opening the UI.                    |
| `cdn.entrypoint` | `<built-in>`               | Override the CDN URL for the dashboard bundle.                                 |
| `cdn.react`      | `https://esm.sh/react`     | CDN URL used to load React.                                                    |
| `cdn.xyflow`     | `https://esm.sh/reactflow` | CDN URL used to load React Flow (graph renderer).                              |

## Authentication & CDN

```ts
new DashboardPlugin({
  auth: {
    enabled: true,
    token: process.env.DASHBOARD_TOKEN,
  },
});
```

Access the dashboard via `http://localhost:3000/dashboard?token=secret` or pass the token in the `Authorization` header. The UI never exposes the token to other clients.

```ts
new DashboardPlugin({
  cdn: {
    entrypoint: 'https://cdn.example.com/frontmcp-dashboard@1.0.0/index.js',
  },
});
```

Use custom CDN URLs when you need offline installs or prefer your own hosting.

## Architecture & data flow

The dashboard serves the HTML page at `GET /dashboard`, connects an SSE stream at `/dashboard/sse`, and makes MCP tool calls under the same base path. The UI calls `dashboard:graph` to fetch the node-and-edge metadata that React Flow renders. When new tools, resources, prompts, or agents register, the SSE stream pushes updates so components redraw immediately.

<Note>
Because the dashboard runs as a standalone app inside the parent scope, the `GraphDataProvider` can enumerate every scope, plugin, and agent without any extra wiring.
</Note>

## MCP tools

### `dashboard:graph`

- Inputs: `includeSchemas?: boolean`, `refresh?: boolean`
- Output: GraphData object with nodes, edges, layout hints, and metadata.

Use this tool when you need a serialized snapshot of the current server graph.

### `dashboard:list-tools`

- Inputs: `filter?: string`, `includeSchemas?: boolean`
- Output: A list of tools with schema metadata suitable for dashboards or observability tooling.

### `dashboard:list-resources`

- Inputs: `filter?: string`, `includeTemplates?: boolean`
- Output: Resources and resource templates that you can map into the graph or audit.
