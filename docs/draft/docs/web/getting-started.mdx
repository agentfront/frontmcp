# Getting Started

This guide walks you through installing and configuring FrontMCP Web for your browser application.

## Prerequisites

**For package manager installation:**
- Node.js 18+ or Bun
- React 18+ (for React integration)
- TypeScript recommended

**For CDN (esm.sh) usage:**
- Modern browser with ES modules support
- No build tooling required

    ### TypeScript Support with esm.sh

    To get full TypeScript support when using esm.sh, you have two options:

    #### Option 1: Type-only installation (Recommended)

    Install the package as a dev dependency for types only:

    ```bash
    npm install @frontmcp/browser --save-dev
    ```

    Then configure your `tsconfig.json` to map the CDN URL to local types:

    ```json
    {
      "compilerOptions": {
        "paths": {
          "https://esm.sh/@frontmcp/browser@*": ["./node_modules/@frontmcp/browser"],
          "https://esm.sh/@frontmcp/browser@*/react": ["./node_modules/@frontmcp/browser/react"],
          "https://esm.sh/@frontmcp/browser@*/store": ["./node_modules/@frontmcp/browser/store"]
        }
      }
    }
    ```

    #### Option 2: Triple-slash reference directive

    esm.sh serves type definitions directly. Add a reference at the top of your file:

    ```typescript
    /// <reference types="https://esm.sh/@frontmcp/browser@0.6.0?dts" />

    import {
      BrowserMcpServer,
      EventTransportAdapter,
    } from 'https://esm.sh/@frontmcp/browser@0.6.0';
    ```

    <Note>
      Some editors like VS Code may require the [Deno extension](https://marketplace.visualstudio.com/items?itemName=denoland.vscode-deno) or similar to resolve remote type references.
    </Note>

## Basic Setup

### 1. Initialize Runtime Config

FrontMCP Web requires runtime configuration for browser environments:

```typescript
import { initializeConfig, generateUUID } from '@frontmcp/browser';

// Initialize once at app startup
initializeConfig({
  debug: import.meta.env.DEV,
  isDevelopment: import.meta.env.DEV,
  machineId: generateUUID(),
});
```

<Warning>
  Call `initializeConfig()` before creating any MCP servers or stores. This is typically done in your app's entry point (e.g., `main.tsx`).
</Warning>

### 2. Create a Transport

Transports handle communication between your app and MCP clients:

```typescript
import { EventTransportAdapter, createSimpleEmitter } from '@frontmcp/browser';

// Create an in-app transport using events
const emitter = createSimpleEmitter();
const transport = new EventTransportAdapter(emitter);
```

<Info>
  For cross-origin communication (iframes, extensions), use `PostMessageTransportAdapter` instead.
</Info>

### 3. Create a Server

The `BrowserMcpServer` is the main entry point:

```typescript
import { BrowserMcpServer } from '@frontmcp/browser';

const server = new BrowserMcpServer({
  name: 'my-app',
  version: '1.0.0',
  transport,
});

// Add a simple tool
server.addTool({
  name: 'greet',
  description: 'Say hello to someone',
  inputSchema: {
    type: 'object',
    properties: {
      name: { type: 'string', description: 'Name to greet' },
    },
    required: ['name'],
  },
  handler: async ({ name }) => ({
    content: [{ type: 'text', text: `Hello, ${name}!` }],
  }),
});

// Start the server
await server.start();
```

### 4. React Integration

Wrap your app with `FrontMcpProvider`:

```tsx
// App.tsx
import { FrontMcpProvider } from '@frontmcp/browser/react';

function App() {
  return (
    <FrontMcpProvider server={server}>
      <YourApp />
    </FrontMcpProvider>
  );
}
```

Now you can use hooks in any component:

```tsx
import { useTool, useMcp } from '@frontmcp/browser/react';

function Greeter() {
  const { isConnected } = useMcp();
  const { execute, loading, error } = useTool('greet');

  const handleGreet = async () => {
    const result = await execute({ name: 'World' });
    console.log(result); // "Hello, World!"
  };

  return (
    <div>
      <p>Status: {isConnected ? 'Connected' : 'Disconnected'}</p>
      <button onClick={handleGreet} disabled={loading}>
        {loading ? 'Loading...' : 'Greet'}
      </button>
    </div>
  );
}
```

## Complete Example

Here's a complete setup with React:

```tsx
// main.tsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import { initializeConfig, generateUUID } from '@frontmcp/browser';
import App from './App';

// Initialize config first
initializeConfig({
  debug: import.meta.env.DEV,
  isDevelopment: import.meta.env.DEV,
  machineId: generateUUID(),
});

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

```tsx
// App.tsx
import { BrowserMcpServer, EventTransportAdapter, createSimpleEmitter } from '@frontmcp/browser';
import { FrontMcpProvider } from '@frontmcp/browser/react';
import { useMemo } from 'react';
import TodoApp from './TodoApp';

export default function App() {
  const server = useMemo(() => {
    const emitter = createSimpleEmitter();
    const transport = new EventTransportAdapter(emitter);

    const srv = new BrowserMcpServer({
      name: 'todo-app',
      version: '1.0.0',
      transport,
    });

    // Add your tools and resources here
    srv.addTool({
      name: 'getCurrentTime',
      description: 'Get the current time',
      handler: async () => ({
        content: [{ type: 'text', text: new Date().toISOString() }],
      }),
    });

    return srv;
  }, []);

  return (
    <FrontMcpProvider server={server} autoStart>
      <TodoApp />
    </FrontMcpProvider>
  );
}
```

## CDN Complete Example

Here's a complete single-file example using esm.sh (no build tools required):

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>FrontMCP Browser Demo</title>
</head>
<body>
  <div id="app">
    <h1>FrontMCP Browser Demo</h1>
    <button id="greet-btn">Greet</button>
    <p id="output"></p>
  </div>

  <script type="module">
    import {
      BrowserMcpServer,
      EventTransportAdapter,
      createSimpleEmitter,
      initializeConfig,
      generateUUID,
    } from 'https://esm.sh/@frontmcp/browser@0.6.0';

    // Initialize config
    initializeConfig({
      debug: true,
      isDevelopment: true,
      machineId: generateUUID(),
    });

    // Create transport and server
    const emitter = createSimpleEmitter();
    const transport = new EventTransportAdapter(emitter);

    const server = new BrowserMcpServer({
      name: 'demo-app',
      version: '1.0.0',
      transport,
    });

    // Add a tool
    server.addTool({
      name: 'greet',
      description: 'Say hello',
      inputSchema: {
        type: 'object',
        properties: {
          name: { type: 'string' },
        },
        required: ['name'],
      },
      handler: async ({ name }) => ({
        content: [{ type: 'text', text: `Hello, ${name}!` }],
      }),
    });

    // Start server
    await server.start();
    console.log('MCP Server started!');

    // Wire up UI
    document.getElementById('greet-btn').addEventListener('click', async () => {
      const result = await server.executeTool('greet', { name: 'World' });
      document.getElementById('output').textContent = result.content[0].text;
    });
  </script>
</body>
</html>
```

Save this as `index.html` and open it in any modern browser - no server required!

## Alternative: Using BrowserScope

For simpler use cases without a full server, use `BrowserScope`:

```typescript
import { createBrowserScope } from '@frontmcp/browser';

const scope = createBrowserScope({
  serverInfo: {
    name: 'simple-app',
    version: '1.0.0',
  },
});

// Register tools directly
scope.registerTool({
  name: 'ping',
  description: 'Ping the server',
  handler: async () => ({ content: [{ type: 'text', text: 'pong' }] }),
});

await scope.start();
```
