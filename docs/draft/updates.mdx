### Features

- Added first-class Vercel KV storage: configure `redis.provider: 'vercel-kv'` for session stores, cache providers, and inspectors plus hybrid Redis pub/sub when you still need subscriptions.
- Introduced `type: 'global-store'` in Cache Plugin so every cached tool automatically reuses the `@FrontMcp`-level store (Redis or Vercel KV) without duplicating credentials.
- Split the UI stack into `@frontmcp/uipack` (HTML components, theming, templates) and `@frontmcp/ui` (widgets, React wrappers) so HTML-first projects stay lightweight.

### Fixes

- Rebuilt the Vercel/Lambda bundle flow with Rspack helpers, saner entry generation, and stronger escaping so serverless deployments ship the same optimized assets as local builds.
- Hardened Handlebars widgets, cache providers, and HTML escaping to stop malformed templates from rendering blank widgets or leaking unsafe output.

### Docs

- Added a dedicated [Vercel KV deployment guide](/docs/deployment/vercel-kv) plus Redis and Serverless callouts covering when to choose Redis vs KV and how to mix them for pub/sub.
- Updated UI library docs, release notes, and import examples to explain the `@frontmcp/ui` / `@frontmcp/uipack` split and new Cache Plugin global-store workflow.

### Build/CI

- Upgraded the workspace to `@modelcontextprotocol/sdk 1.25.1`, Nx 22.3.3, new Rspack/esbuild tooling, and Snyk resolutions so installs remain reproducible on Node 24.
